# Requirements

Languages: Typescript. Optional: Swift, Python
Deployment: Railway

Requirements: 
- Focus: UX, being highly capable, low latency, and natural conversation
- Recall questions/answers from prev session ("What were we talking about yesterday"?)
- Can ingest github urls & answer questions about the repo/comments/issues 

Additional:
- Make it known to user audibly if Jarvis is working 
- Support interruptions (pause when user starts talking again)
- Jarvis can communicate its capabilities/limitations
- Never fabricate or hallucinate. Say "I don't know" if facts are expected
- Can answer questions based on an API (pick one) 

Bonus:
- AWESOME: BUILD APP WITH SWIFT as well
- Can run passively in background, activate on "Jarvis"
- Scale to 10+ simultaneous users; support sign-in with isolateion, allow users to set preferences
- Can open issues, analyze them, open PRs to fix them ("Perform analysis of XYZ issue and open a PR to fix it")
